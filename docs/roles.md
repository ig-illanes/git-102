# Roles dentro del flujo

## Autor del cambio

Persona que crea la rama, realiza el cambio y abre el Pull Request.

Responsabilidades:

- Trabajar en una rama.
- Mantener el cambio enfocado.
- Explicar que cambio y por que.
- Responder comentarios.

## Revisor

Persona que revisa el Pull Request antes de integrarlo.

Responsabilidades:

- Entender el objetivo del cambio.
- Revisar si el cambio es claro y necesario.
- Pedir ajustes si corresponde.
- Aprobar cuando el cambio esta listo.

## Responsable de integracion

Persona que hace merge a `main`, si el equipo tiene ese rol separado.

Responsabilidades:

- Verificar aprobaciones.
- Revisar que no existan conflictos.
- Integrar el cambio aprobado.

## Regla simple

Quien crea el cambio debe explicar.

Quien revisa debe preguntar y validar.

Quien integra debe cuidar la estabilidad de `main`.
